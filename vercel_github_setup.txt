#    دليل إعداد GitHub و Vercel لنشر وتحديث مشروعك تلقائياً      #
##################################################################

مرحباً بك! هذا الدليل سيأخذك خطوة بخطوة خلال عملية نشر تطبيقك على الإنترنت ليصبح متاحاً للجميع عبر رابط ويب.
سنستخدم الأدوات التالية:
- **GitHub:** لاستضافة الكود المصدري لمشروعك وإدارة نسخه.
- **Vercel:** منصة لنشر تطبيقات الويب بسهولة وسرعة، مع ميزة التحديث التلقائي.

العملية مقسمة إلى ثلاث مراحل رئيسية:
1.  **المرحلة الأولى:** رفع الكود الخاص بك إلى مستودع (repository) على GitHub.
2.  **المرحلة الثانية:** ربط هذا المستودع بـ Vercel لنشر الموقع للمرة الأولى.
3.  **المرحلة الثالثة:** فهم كيفية عمل التحديثات التلقائية.

---

### المرحلة الأولى: إعداد GitHub ورفع المشروع

**المتطلبات الأساسية:**
1.  **حساب GitHub:** إذا لم يكن لديك حساب، قم بإنشاء واحد مجاناً من: https://github.com/join
2.  **برنامج Git:** يجب أن يكون مثبتاً على جهازك. إذا لم يكن مثبتاً، قم بتنزيله من: https://git-scm.com/downloads

---

**الخطوة 1: إنشاء مستودع جديد على GitHub**

1.  اذهب إلى صفحتك الرئيسية في موقع GitHub.
2.  في الزاوية العلوية اليمنى، انقر على علامة `+` ثم اختر **"New repository"**.
3.  **اسم المستودع (Repository name):** اختر اسماً مناسباً لمشروعك (مثلاً: `tabuk-health-directory`).
4.  **الوصف (Description):** (اختياري) يمكنك إضافة وصف قصير للمشروع.
5.  اختر **"Public"** ليكون المشروع عاماً ومتاحاً للجميع.
6.  **نقطة مهمة:** **لا تحدد** أي خيار من الخيارات التالية (Add a README file, Add .gitignore, Choose a license). نريد إنشاء مستودع فارغ تماماً.
7.  انقر على زر **"Create repository"**.

---

**الخطوة 2: تهيئة Git ورفع الكود من جهازك**

افتح الطرفية (Terminal أو Command Prompt) ونفذ الأوامر التالية **داخل مجلد مشروعك**:

1.  **الانتقال إلى مجلد المشروع:**
    ```bash
    cd /path/to/your/project-folder
    ```
    (استبدل `/path/to/your/project-folder` بالمسار الفعلي لمجلد مشروعك)

2.  **تهيئة Git في المجلد:**
    ```bash
    git init
    ```

3.  **إضافة جميع ملفات المشروع إلى Git:**
    ```bash
    git add .
    ```

4.  **حفظ النسخة الأولى من المشروع (Commit):**
    ```bash
    git commit -m "Initial commit"
    ```

5.  **تغيير اسم الفرع الرئيسي إلى `main` (ممارسة حديثة موصى بها):**
    ```bash
    git branch -M main
    ```

6.  **ربط مجلدك المحلي بالمستودع على GitHub:**
    من صفحة المستودع التي أنشأتها على GitHub، انسخ السطر الذي يبدأ بـ `git remote add origin`. سيكون شكله كالتالي:
    ```bash
    git remote add origin https://github.com/your-username/your-repository-name.git
    ```
    (استبدل `your-username` و `your-repository-name` بالمعلومات الصحيحة الخاصة بك).
    الصق الأمر في الطرفية واضغط Enter.

7.  **رفع الكود إلى GitHub:**
    ```bash
    git push -u origin main
    ```
    قد يُطلب منك تسجيل الدخول إلى حسابك في GitHub. بعد انتهاء الأمر، قم بتحديث صفحة المستودع على الموقع وستجد كل ملفاتك قد ظهرت هناك.

### المرحلة الثانية: نشر المشروع باستخدام Vercel

**الخطوة 1: إنشاء حساب Vercel وربطه بـ GitHub**

1.  اذهب إلى موقع Vercel: https://vercel.com/signup
2.  اختر **"Continue with GitHub"** لإنشاء حساب وربطه مباشرة بحسابك في GitHub. اتبع خطوات الربط ومنح الأذونات اللازمة.

---

**الخطوة 2: استيراد المشروع إلى Vercel**

1.  من لوحة تحكم Vercel، انقر على **"Add New..."** ثم اختر **"Project"**.
2.  ستظهر لك قائمة بمستودعاتك على GitHub. ابحث عن المستودع الذي أنشأته (`tabuk-health-directory`) وانقر على زر **"Import"** بجانبه.

---

**الخطوة 3: إعدادات النشر**

سيحاول Vercel التعرف على نوع مشروعك. بما أن مشروعنا لا يستخدم أدوات بناء معقدة، سنتأكد من أن الإعدادات صحيحة.

1.  **Project Name:** يمكنك ترك الاسم كما هو أو تغييره.
2.  **Framework Preset:** اتركه على **"Other"**.
3.  **Build and Output Settings:** (هذه هي أهم خطوة)
    - **Build Command:** اضغط على مفتاح **"Override"** واترك الحقل **فارغاً**. مشروعنا لا يحتاج لأمر بناء.
    - **Output Directory:** اضغط على مفتاح **"Override"** واترك الحقل **فارغاً**. ملف `index.html` موجود في المجلد الرئيسي.
    - **Install Command:** اضغط على مفتاح **"Override"** واترك الحقل **فارغاً**.

4.  **Environment Variables (متغيرات البيئة):** لا تحتاج لإضافة أي شيء هنا حالياً.

5.  انقر على زر **"Deploy"**.

---

**الخطوة 4: موقعك مباشر!**

سيبدأ Vercel الآن في نشر موقعك. تستغرق العملية أقل من دقيقة. بعد الانتهاء، سيتم عرض لقطة من موقعك مع تهنئة. انقر على الصورة أو الرابط الذي يمنحك إياه Vercel (مثل: `your-project-name.vercel.app`) لزيارة موقعك المباشر.

**تهانينا! 🎉 موقعك الآن منشور على الإنترنت.**

### المرحلة الثالثة: التحديث التلقائي (CI/CD)

هذه هي أفضل ميزة. الآن بعد أن تم الربط، أي تحديث تقوم به على الكود سيظهر تلقائياً على الموقع المنشور.

**كيفية تحديث موقعك:**

1.  **قم بإجراء التعديلات:** افتح أي ملف في مشروعك على جهازك وقم بالتعديل الذي تريده (مثلاً: تغيير نص، إضافة لون، إصلاح خطأ).

2.  **احفظ التغييرات في Git:** افتح الطرفية مرة أخرى في مجلد مشروعك ونفذ الأوامر التالية:

    - أضف كل الملفات المعدلة:
      ```bash
      git add .
      ```

    - احفظ التغييرات مع رسالة تصف التحديث:
      ```bash
      git commit -m "وصف للتحديث الذي قمت به، مثلاً: تحديث ألوان الأزرار"
      ```

3.  **ارفع التغييرات إلى GitHub:**
    ```bash
    git push origin main
    ```

**وهذا كل شيء!**

بمجرد انتهاء أمر `push`، سيقوم Vercel تلقائياً باكتشاف التغيير، وسيبدأ عملية نشر جديدة. في غضون دقيقة، قم بزيارة رابط موقعك مرة أخرى وستجد أن تعديلاتك قد ظهرت. لست بحاجة لفتح موقع Vercel أبداً للقيام بالتحديث.
